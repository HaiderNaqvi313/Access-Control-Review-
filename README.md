
# 🔐 Access Control Review Project

## 📌 Overview
This project simulates a cybersecurity access control audit to evaluate user permissions and identify risks based on the principle of least privilege.

## 📊 Dataset
- 20 users across multiple departments
- Attributes include:
  - User ID
  - Department
  - Role
  - Access Level
  - Issue
  - Risk Level

## ⚙️ Analysis Performed
- Reviewed user roles and permissions
- Identified overprivileged accounts
- Classified risks into High, Medium, and Low categories

## 📈 Summary of Findings
- Total Users: 20  
- High Risk Issues: 4  
- Medium Risk Issues: 4  
- Low Risk Issues: 0  

### Key Observations:
- Multiple users with excessive privileges (Admin access)
- Cross-department access misalignment
- One account with unrestricted access across systems

## 📊 Visualization
A chart comparing High vs Medium risk users is included in the repository.

## 🚨 Risk Impact
- Unauthorized system access
- Potential data breaches
- Compliance violations

## ✅ Recommendations
- Enforce least privilege access
- Remove unnecessary permissions
- Implement role-based access control (RBAC)
- Conduct periodic access reviews
